I will write a paragraph for each methodology of SDLC. There are many of them but I will suggest a few. My suggestions and reasons are as follows

  • Waterfall Model- This is a Bruteforce method which is very easy to understand. It has 6 stages typically starting from Requirements gathering, designing solutioning and implementation designing, actual implementation of coding, testing it in UAT, installation of the packages at places wherever required for the users to the final stage which is just maintaining it. The cycle is very simple to implement but very tedious as all the requirements should be known. The stage freeze happens when you cross that stage to the next one. So if the requirements are very clear then it is one of the best methods available to follow for Software Development.

  • V Shaped SDLC Model- Lets just say it is an enhanced version of Waterfall model. The additional step here is that it pairs up every development stage with a testing phase to overcome the disadvantages of waterfall model. If your development stage is paired up with testing stage as well then there are less chances of bugs in production or in the final release or you dont have to do one big bang bang release/testing. It works well for the projects which are small and where requirements are known. It doesnt work well where there is more complication in requirements or things change rapidly.

  • Rapid Application Development- As the name suggests its used for developing a project in a fast pace. The project is broken down into modules and important modules are developed first. It follows a Time Box approach where customer is also involved in its stages. Customer involement reduces the risk of the modules to be different from the actual expectations of the customers. It should be used when requirements are known and technical complexities are less. It can also be used for time sensitive projects.

Among so many SDLC methodologies the ones that are in AGILE classification are better. Among AGILE SDLC methods, I also have a few suggestions.

  • Extreme Programming- Its used for small to medium sized projects where there is a lot of coding required. Even communication among teammates is done through coding. It starts with a planning stage and then small small releases are done in the form of different stories and that is used across the teams to communicate and to keep a track of the progress. The requirements are written in the form of test cases as well. It involves a lot of pair programming as well.

  • Feature Driven Design- It involved listing down of features after planning and drawing sequential diagrams from the meetings. Then the development sequence is planned feature by feature. The team produces sequence diagrams feature by feature. And then the development happens by features. Testing and release happens after every feature or a set of feature development. Its the mostly adapted method in the Software Industry. As its a very practical method to keep the dependency less on extrenal factors for example an employee leaving the company.

  • Adaptive Software Development- It involves rapid response to changing requirements. The project is divided into various iterations and then risk analysis, development, testing etc. It bascially has three phases- Speculation- Its the initial phase where basically requirements are gathered through discussions, analysis, surveys etc. Then comes the next phase which is Collaboration- It involves communication and collaboration but individual creativity also plays a major role. Then comes the third phase which is- Learning. Here the workers are expected to learn more about the project, technologies etc through group discussions and technical interviews. It definitely leads to a good quality output but its time consuming and reaching the quality expected is not always possible.

  • Scrum- Its used for complex projects where several teams have to work towards one common goal. Its a framework that team uses to self organize. The various features are Sprint- which is a time box method for development completion. Sprint Review- To check incomplete tasks regarding the features. Sprint Retrospective- Its a phase where project backlogs are addressed and a conclusion is made. The advantages are that its a very well organised framework for any MNC to use. When its combined with Feature development technique, it becomes the most widely adapted technique in the MNCs. Here customer satisfaction is also very important.

I have used CSS in this webpage and there are three ways to use it.

  • First is the inline styling method where you write CSS directly as an attribute to the HTML tag. 

                            <h1 style="color: blue;">Hello World!</h1>

  • Then we can also add a <style> tag in the HTML file to define classes and IDs. After that, use those class names and IDs as attributes in the HTML tags. Here's an example: 

                            <style>
                            .red-text {
                            color: red;
                            }
                            #blue-heading {
                            color: blue;
                            }
                        </style>
                        
                        <h1 id="blue-heading">This is a blue heading</h1>
                        <p class="red-text">This is a red paragraph.</p>

  • The third way is to create a separate CSS file and link it using the <link> tag in the HTML file. Here's an example: 

                                <!-- Link the external CSS file -->
                            <link rel="stylesheet" href="styles.css">
                          
                            <!-- Content styled using external CSS -->
                            <h1 class="green-heading">Welcome!</h1>
                            <p class="italic-text">This is styled using an external CSS file.</p>
                          
                            /* styles.css */
                            .green-heading {
                              color: green;
                            }
                            .italic-text {
                              font-style: italic;
                            }

The reason CSS is used in the webpage are-

  1. To make the page look more beautiful. This amounts to better user experience.

  2. Faster page speed as you can use the same classes and common styling at various places. So less code for more functionalities.

  3. Web page compatibility across browsers and devices. CSS is a great tool to build a page for businessed which can be used by clients and customers in various devices. The page will look organised and beautiful in all the devices and browsers.
